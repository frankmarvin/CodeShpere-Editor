# 🔐 Security Policy

## 🛡 Supported Versions

The following versions of CodeSphere-Editor are currently supported with security updates:

| Version | Supported |
|---------|-----------|
| Latest  | ✅ Yes |
| Older versions | ❌ No |

---

## 🚨 Reporting a Vulnerability

If you discover a security vulnerability in CodeSphere-Editor, please report it responsibly.

### 📩 How to Report

- Email: your-email@example.com  
- Or open a private security advisory on GitHub  
- Do NOT publicly disclose the vulnerability before it is reviewed.

Please include:

- A detailed description of the issue
- Steps to reproduce the vulnerability
- Possible impact
- Screenshots (if applicable)
- Suggested fix (if available)

---

## 🔒 Security Best Practices

CodeSphere-Editor follows these security principles:

- Dependency auditing using `npm audit`
- Minimal external package usage
- Secure environment variable handling
- Regular dependency updates
- Safe in-browser execution sandbox (when enabled)
- No storage of sensitive user credentials by default

---

## 🔄 Response Timeline

- Initial response: within 48 hours  
- Status update: within 5 business days  
- Patch release (if confirmed): as soon as possible  

---

## 🤝 Responsible Disclosure

We appreciate responsible disclosure and will credit contributors who report valid vulnerabilities.

Thank you for helping keep CodeSphere secure.
